# BallPark Lab 시작 가이드

## 목표

첫 목표는 완성도 높은 서비스가 아니라, **보안진단할 수 있는 최소 기능을 가진 Web/API 서비스**를 만드는 것입니다.

## 1단계: 서비스 이해

먼저 다음 질문에 답해보세요.

```text
이 서비스의 사용자는 누구인가?
어떤 데이터가 민감한가?
관리자만 할 수 있어야 하는 기능은 무엇인가?
일반 사용자가 절대 접근하면 안 되는 데이터는 무엇인가?
외부 API 또는 파일 업로드처럼 위험도가 높은 기능은 무엇인가?
```

## 2단계: 자산 목록 작성

| 자산 | 설명 | 중요도 |
|---|---|---|
| 사용자 계정 | 로그인, 권한, 포인트 | High |
| 직관 기록 | 사용자의 활동 기록 | Medium |
| 포인트 | 예매 시뮬레이션 재화 | High |
| 경기 데이터 | 관리자 수정 대상 | Medium |
| 댓글 | 사용자 입력값 | Medium |
| 업로드 파일 | 프로필 이미지 | Medium |

## 3단계: 테스트 범위 정의

이번 1차 진단 범위는 다음으로 제한합니다.

```text
인증/세션
접근제어
댓글 입력값 처리
직관 기록 API
포인트/예매 비즈니스 로직
관리자 API
파일 업로드
```

## 4단계: 첫 번째로 확인할 기능

추천 순서:

1. alice로 로그인한다.
2. `/api/me`로 내 계정 정보를 확인한다.
3. `/api/users/1/attendance`와 `/api/users/2/attendance`의 응답 차이를 확인한다.
4. 일반 사용자 토큰으로 `/api/admin/games/1` 수정 요청이 가능한지 확인한다.
5. 댓글을 작성하고 `/games/1/page`에서 출력 방식을 확인한다.
6. 포인트 충전과 티켓 예매 요청에서 서버가 어떤 값을 신뢰하는지 확인한다.

## 5단계: 보고서 작성 흐름

취약점 하나마다 다음 구조를 사용하세요.

```text
취약점명:
위험도:
영향받는 기능:
취약점 개요:
발생 원인:
비즈니스 영향:
재현 조건:
조치 방안:
재검증 기준:
```

## 6단계: 수정 버전에서 고칠 항목

- 비밀번호 저장 방식을 bcrypt/argon2 계열로 변경
- 토큰 Secret을 환경변수로 이동
- 모든 객체 조회/수정 API에 소유자 검증 추가
- 관리자 API에 role 검사 추가
- HTML 출력 시 이스케이프 적용
- 포인트/예매 가격은 서버에서 계산
- 중복 좌석 예약 방지
- 파일 업로드 확장자, 크기, MIME Type 검증
- 로그인/댓글/신고/검색 기능에 Rate Limit 적용
